1 Description du système
1.1 Contexte

De nombreux jeunes talents acquièrent des compétences précieuses à travers des mis-
sions ponctuelles, stages, projets ou emplois temporaires. Cependant, ces expériences sont

souvent difficiles à valoriser dans un CV classique, car elles manquent de preuves véri-
fiables.

Par ailleurs, les outils RH existants sont souvent lourds, coûteux ou inadaptés aux
petites structures et aux jeunes profils.
1.2 Description générale
Passerelle RH est une application web RH simplifiée permettant aux utilisateurs de :
— Créer et gérer leurs expériences professionnelles (missions)
— Obtenir des preuves professionnelles validées par un employeur ou un responsable
— Générer des attestations PDF avec QR code
— Présenter un profil professionnel fiable et structuré

Le système s’inspire des principes classiques de la gestion RH (validation, suivi, sta-
tistiques), tout en apportant une dimension innovante orientée employabilité.

2 Objectifs du système
2.1 Objectifs principaux
— Mettre en place une plateforme RH légère et sécurisée
— Permettre la création et validation d’expériences professionnelles
— Générer des preuves vérifiables (PDF + QR code)
— Centraliser les missions validées dans un profil professionnel unique
— Garantir la traçabilité et la fiabilité des validations
2.2 Objectifs secondaires
— Offrir une visualisation claire des données RH via un dashboard
— Améliorer la visibilité des compétences des utilisateurs
— Introduire un score de fiabilité simple basé sur les validations
— Faciliter l’évolution vers une version plus avancée (V2)
— Appliquer les bonnes pratiques de développement full stack

3 Utilisateurs du système
3.1 Utilisateur (Jeune / Employé)
Rôle : USER
— Créer et gérer son profil
— Ajouter des missions professionnelles
— Générer des liens de validation
— Consulter ses attestations et son score
— Exporter ses preuves professionnelles
3.2 Validateur (Employeur / Responsable)
Rôle : VALIDATOR
— Accéder à une mission via un lien sécurisé
— Valider ou refuser une mission
— Donner une évaluation simple (notes + commentaire)
3.3 Administrateur
Rôle : ADMIN
— Gérer les utilisateurs
— Superviser les missions et validations
— Consulter les statistiques globales
— Gérer les référentiels (compétences, badges)
— Modérer les contenus si nécessaire

4 Fonctionnalités fonctionnelles
4.1 Authentification et autorisation
— Inscription et connexion sécurisées
— Authentification JWT (access + refresh token)
— Gestion des rôles (ADMIN / USER / VALIDATOR)
— Protection des routes backend et frontend
— Déconnexion sécurisée

4.2 Gestion des profils utilisateurs
— Création et mise à jour du profil

5.1 Backend
— Langage : Java 21
— Framework : Spring Boot
— Sécurité : Spring Security + JWT
— ORM : Spring Data JPA / Hibernate
— Base de données : PostgreSQL
— Documentation API : Swagger / OpenAPI
— PDF : OpenPDF
— QR Code
— Mapping : MapStruct
— Tests : JUnit 5, Mockito
5.2 Frontend
— Framework : Angular 17 ou React
— UI : Tailwind CSS
— Graphiques : Chart.js
— Responsive
5.3 Architecture
— Architecture en couches : Controller → Service → Repository
— Communication REST (JSON)
— DTOs pour échanges sécurisés
— Séparation claire des responsabilités

6 Conclusion
Passerelle RH est un projet fil rouge simple, innovant et parfaitement adapté à une
réalisation sur 3 mois. Il combine les bases solides d’un système RH classique avec une
approche moderne orientée employabilité, tout en respectant les contraintes académiques
et techniques de Youcode
